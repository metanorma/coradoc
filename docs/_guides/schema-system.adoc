= Schema System Guide

== Task

Use Coradoc's schema system to handle specialized document types like ISO standards, technical specifications, or other structured documents.

== Approach

The schema system allows you to:

1. Define custom parsing rules for specific document types
2. Specify custom serialization formats
3. Enforce document structure constraints
4. Handle type-specific attributes and metadata

== Understanding Schemas

A schema in Coradoc defines:

* **Parsing rules** - How to parse specific document types
* **Serialization rules** - How to serialize for specific output formats
* **Structure constraints** - Required/optional elements
* **Attribute definitions** - Document metadata

== Using Built-in Schemas

=== ISO Schema

Parse ISO-standard documents:

[source,ruby]
----
doc = Coradoc.parse_file("iso-document.adoc", schema: :iso)
----

The ISO schema handles:

* ISO-specific front matter
* Standard numbering conventions
* ISO metadata attributes
* Clause structures

=== Default Schema

The default schema is used when no schema is specified:

[source,ruby]
----
# These are equivalent
doc = Coradoc.parse(input)
doc = Coradoc.parse(input, schema: :default)
----

## Creating Custom Schemas

=== Step 1: Define the Schema Class

[source,ruby]
----
module Coradoc
  module Schema
    class CustomSchema < Schema
      def schema_name
        :custom
      end

      def rules
        # Define parsing and serialization rules
      end
    end
  end
end
----

=== Step 2: Define Parsing Rules

[source,ruby]
----
module Coradoc
  module Schema
    class CustomSchema < Schema
      def parsing_rules
        {
          front_matter: ->(node) { parse_front_matter(node) },
          metadata: ->(node) { parse_metadata(node) }
        }
      end

      private

      def parse_front_matter(node)
        # Custom front matter parsing logic
      end

      def parse_metadata(node)
        # Custom metadata parsing logic
      end
    end
  end
end
----

=== Step 3: Define Serialization Rules

[source,ruby]
----
module Coradoc
  module Schema
    class CustomSchema < Schema
      def serialization_rules
        {
          front_matter: ->(doc) { serialize_front_matter(doc) },
          metadata: ->(doc) { serialize_metadata(doc) }
        }
      end

      private

      def serialize_front_matter(doc)
        # Custom front matter serialization logic
      end

      def serialize_metadata(doc)
        # Custom metadata serialization logic
      end
    end
  end
end
----

=== Step 4: Register the Schema

[source,ruby]
----
Coradoc::Schema.register(:custom, Coradoc::Schema::CustomSchema)
----

=== Step 5: Use the Schema

[source,ruby]
----
doc = Coradoc.parse_file("document.adoc", schema: :custom)
----

## Schema Attributes

Schemas can define custom document attributes:

[source,ruby]
----
module Coradoc
  module Schema
    class TechSpecSchema < Schema
      def default_attributes
        {
          "doctype" => "technical-specification",
          "status" => "draft",
          "version" => "1.0"
        }
      end
    end
  end
end
----

These attributes are automatically added to documents parsed with this schema.

## Schema Constraints

Enforce document structure constraints:

[source,ruby]
----
module Coradoc
  module Schema
    class StrictSchema < Schema
      def required_sections
        %w[introduction scope definitions references]
      end

      def validate_document(doc)
        missing = required_sections - doc.sections.map { |s| s.title.content.first.to_s.downcase }
        unless missing.empty?
          raise "Missing required sections: #{missing.join(', ')}"
        end
      end
    end
  end
end
----

## Complete Example: Technical Report Schema

[source,ruby]
----
require 'coradoc'

module Coradoc
  module Schema
    class TechReportSchema < Schema
      def schema_name
        :tech_report
      end

      def default_attributes
        {
          "doctype" => "technical-report",
          "org" => "Acme Corporation",
          "confidentiality" => "public"
        }
      end

      def required_sections
        %w[abstract introduction methodology results conclusions]
      end

      def validate_document(doc)
        missing = required_sections - doc.sections.map do |s|
          s.title&.content&.first&.to_s&.downcase || ""
        end
        unless missing.empty?
          warn "Warning: Missing recommended sections: #{missing.join(', ')}"
        end
      end

      def pre_parse(input)
        # Add custom front matter if missing
        unless input.include?(':doctype:')
          input = ":doctype: technical-report\n\n#{input}"
        end
        input
      end

      def post_serialize(output)
        # Add custom footer
        output + "\n\n----\nGenerated by Acme Tech Report Generator\n"
      end
    end
  end
end

# Register the schema
Coradoc::Schema.register(:tech_report, Coradoc::Schema::TechReportSchema)

# Use the schema
doc = Coradoc.parse_file("report.adoc", schema: :tech_report)

# Validate
doc.schema.validate_document(doc)

# Serialize with schema post-processing
output = doc.schema.post_serialize(doc.to_adoc)
puts output
----

## Schema Architecture

=== Schema Base Class

[source,ruby]
----
module Coradoc
  module Schema
    class Schema
      attr_reader :name, :options

      def initialize(options = {})
        @options = options
      end

      def schema_name
        raise NotImplementedError
      end

      def default_attributes
        {}
      end

      def parsing_rules
        {}
      end

      def serialization_rules
        {}
      end

      def validate_document(doc)
        # Override in subclass
      end
    end
  end
end
----

=== Schema Registry

[source,ruby]
----
module Coradoc
  module Schema
    class Registry
      def self.register(name, schema_class)
        schemas[name] = schema_class
      end

      def self.get(name)
        schemas.fetch(name) { raise "Unknown schema: #{name}" }
      end

      def self.schemas
        @schemas ||= {}
      end
    end
  end
end
----

## Troubleshooting

**Problem:** Schema isn't being applied.

**Solution:** Verify the schema is registered and the name matches exactly. Check `Coradoc::Schema.registry.keys`.

**Problem:** Document validation fails.

**Solution:** Ensure all required sections are present with correct titles. Check case sensitivity in title matching.

**Problem:** Custom attributes aren't showing up.

**Solution:** Verify `default_attributes` returns a hash. Check that attributes don't conflict with document-level attributes.
