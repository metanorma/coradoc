= HTML to AsciiDoc Conversion Tutorial

== Purpose

This tutorial teaches you how to convert HTML documents to AsciiDoc using Coradoc's HTML input converter.

== Prerequisites

* Completed link:basic-parsing[Basic Parsing Tutorial]
* Basic understanding of HTML structure
* Familiarity with AsciiDoc syntax

== Tutorial Overview

You'll learn to:

* Convert HTML to AsciiDoc
* Handle common HTML elements
* Customize the conversion process
* Handle complex HTML structures

== HTML Input Converter

Coradoc includes an HTML input converter that transforms HTML into the Document Model, which can then be serialized to AsciiDoc.

[source,ruby]
----
require 'coradoc'

html = "<h1>Title</h1><p>Content here.</p>"

# Parse HTML to Document Model
doc = Coradoc::Input::HTML.convert(html)

# Serialize to AsciiDoc
asciidoc = doc.to_adoc
----

== Example 1: Basic HTML Conversion

=== The Code

[source,ruby]
----
require 'coradoc'

html = <<~HTML
  <!DOCTYPE html>
  <html>
  <head><title>My Document</title></head>
  <body>
    <h1>Main Title</h1>
    <p>This is a paragraph.</p>
  </body>
  </html>
HTML

# Convert HTML to Document Model
doc = Coradoc::Input::HTML.convert(html)

# Serialize to AsciiDoc
puts "Converted AsciiDoc:"
puts doc.to_adoc
----

=== What It Does

1. Parses HTML document
2. Converts to Coradoc Document Model
3. Serializes to AsciiDoc

=== Expected Output

[source,asciidoc]
----
Converted AsciiDoc:
= Main Title

This is a paragraph
----

== HTML Conversion Reference

### Supported Elements

| HTML Element | AsciiDoc Equivalent |
|-------------|-------------------|
| `<h1>` - `<h6>` | `=` heading levels |
| `<p>` | Paragraph |
| `<ul>`, `<li>` | Unordered list (`*`) |
| `<ol>`, `<li>` | Ordered list (`.`) |
| `<strong>` | Bold (`*text*`) |
| `<em>` | Italic (`_text_`) |
| `<code>` | Monospace (`` `text` ``) |
| `<a>` | Link |
| `<table>` | Table |
| `<pre><code>` | Listing block |

### Unsupported Elements

Some HTML elements may not convert cleanly:

* `<div>` - Often converted to paragraphs or ignored
* `<span>` - May not preserve semantic meaning
* Complex nested structures
* Script and style elements

== Customizing Conversion

The HTML converter uses a plugin system for handling specific elements. You can customize conversion by:

### 1. Creating Custom Converters

[source,ruby]
----
module Coradoc
  module Input
    module HTML
      module Converter
        class CustomConverter < Base
          def convert(node)
            # Custom conversion logic
          end
        end
      end
    end
  end
end
----

### 2. Registering Converters

[source,ruby]
----
Coradoc::Input::HTML.register_converter(
  :custom_element,
  Coradoc::Input::HTML::Converter::CustomConverter
)
----

## Complete Example: Web Page Conversion

=== The Code

[source,ruby]
----
require 'coradoc'

html = <<~HTML
  <!DOCTYPE html>
  <html>
  <head>
    <title>My Article</title>
  </head>
  <body>
    <article>
      <h1>Understanding Round-tripping</h1>

      <p>Round-tripping is the process of <strong>parsing</strong> a document,
      <em>serializing</em> it, and <code>parsing</code> it again.</p>

      <h2>Key Concepts</h2>

      <ul>
        <li>Semantic preservation</li>
        <li>Structure equivalence</li>
        <li>Content fidelity</li>
      </ul>

      <h2>Example</h2>

      <pre><code># Parse and serialize
doc = Coradoc.parse(input)
output = doc.to_adoc</code></pre>

      <p>See <a href="https://example.com">the documentation</a> for more.</p>
    </article>
  </body>
  </html>
HTML

doc = Coradoc::Input::HTML.convert(html)
puts "Converted AsciiDoc:"
puts doc.to_adoc
----

=== Expected Output

[source,asciidoc]
----
Converted AsciiDoc:
= Understanding Round-tripping

Round-tripping is the process of *parsing* a document,
_serializing_ it, and `parsing` it again.

== Key Concepts

* Semantic preservation
* Structure equivalence
* Content fidelity

== Example

Parse and serialize code:

  # Parse and serialize
  doc = Coradoc.parse(input)
  output = doc.to_adoc

See https://example.com[the documentation] for more.
----
